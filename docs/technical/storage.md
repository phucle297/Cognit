# Storage

Cognit is local-first and stores everything under a single `.cognit/`
directory at the project root. There is no remote component.

## Locating the project root

`apps/cli/src/paths.ts:10` (`findProjectRoot`) walks up from
`process.cwd()` (or the value of `--root` / `$COGNIT_ROOT`) looking for a
directory containing `.cognit/cognit.yaml`. It returns the directory
containing `.cognit/` or `null` if no project is found. The walk stops at
the filesystem root (`path.parse(current).root`).

## `.cognit/` directory layout

All paths are built by `projectPaths()` in `apps/cli/src/paths.ts:66`. The
constants at `paths.ts:39-48` enumerate every directory and file the
project owns.

```
.cognit/
├── cognit.yaml              # project config (Schema-validated)
├── cognit.db                # SQLite event store (WAL mode)
├── .gitignore               # ignores db + runtime state, commits config
├── current-session          # plain text — id of the active session
├── current-session.tmp      # atomic-write target for current-session
├── inbox/                   # external publishers drop JSON envelopes here
│   └── _error/              # failed envelopes + <name>.reason.txt sidecars
├── processed/               # successfully processed envelopes (sibling of inbox/)
├── artifacts/
│   └── curated/             # curated artifacts attached to sessions
├── snapshots/               # frozen SessionState projections (json)
└── archive/                 # archived artifacts past cleanup thresholds
```

### Files (constants at `paths.ts:48`)

| File                  | Constant (`paths.ts`) | Purpose                                                                  |
|-----------------------|------------------------|--------------------------------------------------------------------------|
| `cognit.yaml`         | `config`               | Validated project config (see [configuration.md](./configuration.md)).                  |
| `.gitignore`          | `gitignore`            | Generated by `cognit init`; ignores db + runtime state, commits config. |
| `cognit.db`           | `db`                   | SQLite database — the event store itself.                                |
| `current-session`     | `currentSession`       | Plain-text id of the active session for this shell.                      |
| `current-session.tmp` | `currentSessionTmp`    | Atomic-write target — renamed to `current-session` when ready.           |

### Directories (constants at `paths.ts:39`)

| Directory                | Constant (`paths.ts`)  | Purpose                                                                       |
|--------------------------|-------------------------|-------------------------------------------------------------------------------|
| `.cognit/inbox/`         | `inbox`                 | Drop folder for JSON envelopes from external CLIs. See [hooks/README.md](../hooks/README.md).        |
| `.cognit/inbox/_error/`  | `inboxError`            | Failed envelopes land here with a `.reason.txt` sidecar.                       |
| `.cognit/processed/`     | `processed`             | Successfully processed envelopes (renamed to `<event-id>.json`).               |
| `.cognit/artifacts/`     | `artifacts`             | Content-addressed files attached to events (`artifacts` table, sha256-indexed).|
| `.cognit/artifacts/curated/` | `artifactsCurated`  | Curated subset shown in the dashboard's Recovery / Knowledge Graph views.      |
| `.cognit/snapshots/`     | `snapshots`             | Frozen `SessionState` JSON projections, one per snapshot event.                |
| `.cognit/archive/`       | `archive`               | Artifacts older than `cleanup.artifact_max_age_days` and not referenced.       |

## SQLite layout

The database is a single SQLite file at `.cognit/cognit.db` (`paths.ts:72`).
The schema is applied via the DDL in `packages/db/src/schema/tables.ts:11-129`
(see [data-model.md](./data-model.md) for the per-table breakdown). Open pragmas are at
`tables.ts:132`:

- `journal_mode = WAL` — crash-safe + concurrent reader (so the CLI and
  the Hono server can share the file).
- `synchronous = NORMAL` — fsync at commit; WAL handles the rest.
- `foreign_keys = ON` — referential integrity is real.
- `busy_timeout = 5000` — 5s tolerance for short-lived lock contention.

Schema migrations are tracked in the singleton `schema_version` table
(`tables.ts:109`) and applied by `packages/db/src/migrate.ts`.

## Backup + portability

- The full project can be exported to a `tar.gz` bundle by
  `cognit export` (`apps/cli/src/commands/export.ts:1`); bundle contents:
  manifest + `cognit.yaml` + `cognit.db` + optional `artifacts/`.
- A bundle can be re-imported with `cognit import`
  (`apps/cli/src/commands/import.ts:1`) under one of three policies:
  `skip` / `overwrite` / `fork`.
- Per-session snapshots are stored under `snapshots/` and in the `snapshots`
  table; the reducer folds snapshot + tail on cold-start
  (`packages/core/src/reducer.ts:13`).
