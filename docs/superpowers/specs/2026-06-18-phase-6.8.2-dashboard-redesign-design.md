# Phase 6.8.2 — Dashboard redesign: design spec

**Date:** 2026-06-18
**Bead:** Cognit-8ix.9 (to be created)
**Status:** approved by user
**Approach:** A — shadcn-strict (extend current tokens + components)

## Context

The Phase 6 dashboard ships functional pages (Login, Overview, Timeline,
Knowledge Graph, Decision Graph, Verification, Recovery, Settings) on
top of a hand-rolled shadcn-style component library (Button, Card,
Dialog, Input, Select, Tabs, Tooltip, Badge). User feedback: the UI
is "ugly." The fix is not a rewrite — the existing tokens, components,
and page architecture are sound. The work is to (a) tighten the
design tokens so spacing, typography, and color are consistent, (b)
fill the empty/loading/error-state gaps that every list page has, (c)
add subtle motion, and (d) move the navigation chrome from a top
bar to a sidebar so the data-heavy pages have more horizontal room.

The stack stays hand-rolled shadcn-style. No shadcn CLI, no Radix
Themes, no new runtime deps.

## Goals (acceptance criteria)

1. **Tokens consistency.** Every spacing, color, radius, typography
   value used in the dashboard comes from the `@theme` block in
   `apps/dashboard/src/app/index.css`. No inline magic numbers, no
   ad-hoc `text-[13px]`. Tokens have light + dark parity.
2. **Empty / loading / error states.** Every list page renders a
   meaningful `<EmptyState />` when 0 items, `<Skeleton />` during
   fetch, `<ErrorState />` on network failure. No raw "No data" text.
3. **Motion.** Route enter (200 ms fade + 8 px slide-up), card
   hover (120 ms shadow + scale 1.005), accordion / collapsible
   (320 ms ease-out), sidebar collapse (200 ms width). Honors
   `prefers-reduced-motion`.
4. **Sidebar nav.** Replaces the current top nav. Sticky left rail
   on `lg+`, icon + label. Collapsible to icon-only on `md`. Top bar
   keeps only breadcrumb + user menu.
5. **Status colors.** Session and event statuses (`active`,
   `pending`, `failed`, `verified`, `archived`, `neutral`) get
   consistent token-backed colors used by `<StatusPill />` and
   `<Badge variant="…" />`.
6. **Bundle budget.** `dist/` size ≤ 500 KB gzipped (current
   ~300 KB; the new components must not push past 500 KB).
7. **Test count.** Component unit tests for every new shared
   component (Button variants, StatusPill, EmptyState, Skeleton,
   StatCard). Test count grows by ≥ 12. Project test total lands
   in the 567–580 range (current 555).
8. **Visual parity.** Every page screenshot in light + dark
   matches the agreed design (manual review against this spec).

## Non-goals (explicit out of scope)

- Accessibility deep pass (keyboard nav audit, screen-reader test,
  WCAG AA re-cert). The 401 fix added in 6.8.1 covers HttpOnly
  cookies, but no other a11y work. Deferred to phase 6.8.3.
- Mobile responsive beyond the sidebar collapse. Pages still target
  `md+`.
- i18n.
- New backend routes.
- Server bundle changes (already shipped in 6.8.1).

### Design tokens

All tokens live in `apps/dashboard/src/app/index.css` under
`@theme` + `@media (prefers-color-scheme: dark)`. Tailwind 4's
CSS-first config picks them up automatically.

#### Typography

```
--font-sans: ui-sans-serif, system-ui, -apple-system, "Segoe UI", Roboto, "Inter var", sans-serif
--font-mono: ui-monospace, "JetBrains Mono", "SF Mono", Menlo, monospace
Sizes (existing Tailwind scale): xs(0.75) sm(0.875) base(0.9375) lg(1.0625) xl(1.25) 2xl(1.5) 3xl(1.875)
Line-heights: tight(1.2) snug(1.35) normal(1.5) relaxed(1.625)
```

Page heading uses `text-2xl font-semibold tracking-tight leading-tight`.
Section heading uses `text-lg font-semibold`. Body uses `text-sm
leading-normal`. Mono used only for IDs, types, command output.

#### Spacing

Tailwind 4-px base scale stays. Add semantic aliases:

```
--space-page-x: 1.5rem     /* page horizontal padding */
--space-page-y: 2rem       /* page vertical padding */
--space-card:  1.5rem      /* card inner padding */
--space-stack: 1rem        /* vertical stack between siblings */
```

Used as: `px-[length:var(--space-page-x)]` in AppShell, `p-[length:var(--space-card)]` in Card.

#### Radius

```
--radius-sm:  0.25rem   /* badges, inputs, tooltips */
--radius:     0.5rem    /* buttons, cards (default) */
--radius-lg:  0.75rem   /* modals, sheets */
--radius-xl:  1rem      /* hero cards */
```

#### Color refinements

Status palette (semantic, used by StatusPill + Badge):

```
--color-status-active:    oklch(0.65 0.18 145)   /* green */
--color-status-pending:   oklch(0.72 0.16 75)    /* amber */
--color-status-failed:    oklch(0.60 0.22 25)    /* red */
--color-status-verified:  oklch(0.60 0.18 250)   /* blue */
--color-status-archived:  oklch(0.55 0.01 250)   /* gray */
```

For each: a `-bg` variant at 12% chroma (subtle fill) and a `-fg`
variant at full chroma (text/border). Implementation: a `Status`
helper in `shared/config/status.ts` maps a status string → `{ bg, fg, label, icon }`.

Existing color tokens (`--color-background`, `--color-foreground`,
`--color-muted`, etc.) stay as-is. Dark-mode overrides get the
status-color counterparts for parity.

#### Elevation

```
--shadow-sm: 0 1px 2px oklch(0 0 0 / 0.05)
--shadow:    0 1px 3px oklch(0 0 0 / 0.08), 0 1px 2px oklch(0 0 0 / 0.04)
--shadow-md: 0 4px 12px oklch(0 0 0 / 0.08), 0 2px 4px oklch(0 0 0 / 0.04)
```

#### Motion

```
--ease-out:     cubic-bezier(0.16, 1, 0.3, 1)
--duration-fast: 120ms
--duration-base: 200ms
--duration-slow: 320ms
```

Applied via a `motion.ts` helper that exports a `transition` class
builder: `transition("colors", "fast")` → `"transition-colors
duration-[120ms] ease-[var(--ease-out)]"`. Helper avoids hand-written
duration tokens in every component.

Reduced motion: a top-level CSS rule

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

### Layout chrome

#### `widgets/app-shell` (rewrite)

Two-column on `lg+` (sidebar + main), stacked on smaller:

```
+---------------------------------------+
|  Sidebar  |  NavBar (breadcrumb, user)|
|           |  ----------------------   |
|  nav      |  Outlet (page)            |
|  items    |                           |
|           |                           |
+---------------------------------------+
```

- `grid lg:grid-cols-[14rem_1fr]` (sidebar expanded) or
  `[3.5rem_1fr]` (collapsed).
- Sidebar state lives in a `SidebarProvider` (React context) so
  page components can read it (e.g. for empty-state side panel).

#### `widgets/sidebar` (new, replaces top nav)

- Logo at top: lucide `Boxes` icon + "Cognit" wordmark.
- Sections: `Main` (Overview, Timeline), `Explore` (Knowledge
  Graph, Decision Graph, Verification), `Admin` (Recovery, Settings).
- Each item: `lucide` icon (size 16) + label. Active state: bg
  `accent` + text `accent-foreground`. Inactive: text
  `muted-foreground` hover `accent/50`.
- Footer: collapse toggle (lucide `PanelLeftClose` /
  `PanelLeftOpen`) + version string.
- Persists collapsed state in `localStorage` (key: `cognit.sidebar.collapsed`).

#### `widgets/nav-bar` (rewrite, smaller)

- Breadcrumb (left) + user menu (right: avatar circle + "Sign out"
  dropdown). Height shrinks from `h-14` to `h-12`.

### Pages (per-page plan)

Each page is touched once. Plan order matches the redesign risk
(high → low) so reviewers can see the polished version early.

1. **Login** (`features/auth/login` + `pages/Login.tsx`)
   - Centered card (max-w-sm). Logo + "Sign in" heading + token
     input + submit button + helper text ("Paste your
     COGNIT_API_TOKEN").
   - Background: subtle radial gradient from `--color-background`
     to `--color-muted`.
2. **Overview** (`pages/overview.tsx`)
   - Header: project name (h2) + "New Session" button (primary).
   - Stats row: 3 `StatCard` (Sessions, Events this week,
     Open Decisions). Each shows value + delta vs. last week.
   - Sessions list: `DataTable` (goal, status, last activity,
     actions). Empty state when no sessions.
3. **Timeline** (high priority) (`pages/timeline.tsx`)
   - Sticky header: session goal + `StatusPill` + breadcrumb.
   - Filters: type multi-select, actor multi-select, date range.
   - Body: `DataTable` of events. Each row: time (mono), type
     (colored Badge), actor, summary, expand caret. Click → side
     `Sheet` (right) shows full event payload.
   - Empty state when no events match filter.
4. **Knowledge Graph** (`pages/knowledge-graph.tsx`)
   - Full-bleed xyflow. Sidebar collapses by default here to
     maximise canvas.
   - Refined node theme: rounded `--radius`, soft `--shadow-sm`,
     border by entity type color. Edge color by relationship.
   - Right `Sheet` on node click: title, type, summary, related
     events.
5. **Decision Graph** (`pages/decision-graph.tsx`) — same pattern
   as Knowledge Graph but with conclusion/decision entity theme.
6. **Verification** (`pages/verification.tsx`)
   - `DataTable` (id, command, type, status, duration, actions).
   - Row click → expand inline: stdout / stderr / linked
     hypothesis. Use `Accordion` (radix-based, new shared
     component if not yet in stack).
7. **Recovery Center** (`pages/recovery-center.tsx`)
   - `DataTable` of recovery ops. Action button per row opens a
     `Dialog` (existing) with confirmation.
8. **Settings** (`pages/settings.tsx`)
   - Polish: sectioned cards (Auth, Project, Display). Form
     inputs aligned. Save button (disabled until dirty).

### Cross-cutting

#### Empty / loading / error

Every list page wraps its data fetch:

```tsx
if (error) return <ErrorState message={error.message} onRetry={refetch} />;
if (loading) return <Skeleton variant="rect" className="h-64" />;
if (items.length === 0) return <EmptyState icon={Inbox} title="No sessions yet" description="Start one to see it here." action={<Button>New session</Button>} />;
return <DataTable … />;
```

This is the canonical pattern. Any new page copies it.

#### Motion

- Route enter: `app/main.tsx` wraps `<Outlet />` in a div with
  `animate-[fade-in-up_200ms_var(--ease-out)]`. CSS keyframe
  defined once in `app/index.css`.
- Card hover: `hover:shadow-[var(--shadow)] hover:scale-[1.005] transition-all duration-[var(--duration-fast)] ease-[var(--ease-out)]`.
- Accordion: rely on Radix `Accordion` defaults, override with
  `--duration-slow`.
- Sidebar collapse: `transition-[width] duration-[var(--duration-base)] ease-[var(--ease-out)]`.

Reduced-motion media query (above) suppresses all of this.

## Implementation order (phases for the plan)

The plan that follows this spec sequences implementation in 5
phases. Each phase is independently testable and reviewable.

1. **Tokens + motion** (foundation). `@theme` block + `motion.ts`
   helper + reduced-motion rule. No visible UI change yet, but
   every later phase builds on these.
2. **Shared components**. New components + Button/Card/Badge
   variant additions + StatusPill + tests. No page changes.
3. **AppShell + Sidebar**. Two-column layout + sidebar widget.
   Pages render unchanged inside the new chrome.
4. **Pages** (Login → Overview → Timeline → Graphs → Verification →
   Recovery → Settings). Each page adopts the empty/loading/error
   pattern, uses DataTable where applicable, gets the per-page
   polish from the per-page plan above.
5. **Polish + manual review**. Light/dark screenshots, bundle
   size check, axe-core quick scan (no full a11y pass — deferred).

Each phase is one PR.

## Risks

- **Bundle bloat.** New components + xyflow can push past the
  500 KB budget. Mitigation: code-split heavy pages with
  `React.lazy` (route-level). DataTable is intentionally small
  (no virtualization in v1 — Timeline row counts stay under
  ~500 in the demo).
- **Sidebar breaks xyflow full-bleed.** Knowledge/Decision Graph
  want the whole viewport. Solution: sidebar auto-collapses on
  those routes, with a manual override.
- **Reduced-motion regressions.** Pages that animate outside the
  shared motion helper will be missed. Mitigation: lint rule
  (custom eslint check) warns on `transition-` classes that
  don't go through `motion.ts`. Out of scope for v0.1 — note
  in plan.
- **Token rename breaks dark mode.** Adding status colors
  requires 4 new dark variants. Mitigation: write the
  `@media (prefers-color-scheme: dark)` block alongside the
  light tokens in the same commit.

## Testing

- **Unit (Vitest + Testing Library).** Every new shared
  component gets a `*.test.tsx` next to it. ≥ 12 new tests.
  Existing 555 + 12 = 567 minimum.
- **Visual.** Manual screenshot review for each page in light
  + dark. Diffed against the agreed design.
- **Build budget.** `vite build` output size logged in
  `docs/phase-6.8.2-results.md`; must be ≤ 500 KB gzipped.
- **Smoke.** `docker compose up -d` → login → overview loads
  → click session → timeline loads → click event → side sheet
  opens. Manual but scripted in the results doc.
